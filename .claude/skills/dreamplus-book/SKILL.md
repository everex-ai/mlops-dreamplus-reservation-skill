# 드림플러스 회의실 예약

원하는 조건으로 예약하고, 불가하면 가까운 빈 회의실을 제안한다.
공통 절차는 `docs/skill-runtime.md`를 따른다. **쓰기 작업이므로 실행 전 반드시 사용자 확인.**

## 절차

1. **프리플라이트** — 로그인 확인(미로그인 안내 후 중단).
2. **요청 파악** — 회의실(또는 정원/장비 조건), 날짜(기본 오늘), 시작 시간, 종료(기본 시작+30분),
   미팅명(title). 회의실을 조건으로만 말했으면 `data/rooms.catalog.json`에서 후보를 고른다.
3. **데이터 가져오기** — `docs/skill-runtime.md`의 **"요청 시간대에 겹치는 예약(book용)"** fetch로
   그 시간대에 찬 방만(전 회의실) 받는다. 안 겹치는 방은 모두 비어있으므로 **모든 층에 걸쳐 대안이
   정확**하다. (층별 스코프는 다른 층 방을 잘못 "빈 방"으로 제안할 수 있으니 쓰지 않는다.)
4. **판단**:

   ```bash
   echo '<json>' | node bin/book.mjs <회의실> <시작 HH:mm> [종료 HH:mm] --date <YYYY.MM.DD> --title '<미팅명>' --myid <myId>
   ```

   - 🟢 가능: 마지막 줄 `@@ACTION@@ book {payload}` 출력.
   - 🔴 불가: 가까운 빈 회의실 후보 + 최상위 후보의 `@@ACTION@@ book` 출력.
5. **사용자 확인** — 회의실·날짜·시간·차감 포인트를 요약해 보여주고 **예약할지 확인**받는다.
   불가였다면 제안된 대안 중 무엇으로 예약할지 확인.
6. **예약 실행(브라우저)** — 확인되면 `docs/skill-runtime.md`의 "예약 생성" in-page fetch를
   `@@ACTION@@ book`의 payload로 실행. 응답 `code==="200"`이면 완료 안내.
   (다른 대안으로 예약하려면 그 회의실로 4번을 다시 실행해 payload를 얻는다.)
