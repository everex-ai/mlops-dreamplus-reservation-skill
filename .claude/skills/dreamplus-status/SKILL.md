# 드림플러스 예약 현황 조회

날짜·층 등으로 회의실 예약 현황을 멀티 타임바 그리드로 보여준다.
공통 절차는 `docs/skill-runtime.md`를 따른다. (토큰은 브라우저 안에서만 사용)

## 절차

1. **프리플라이트** — `docs/skill-runtime.md`의 로그인 확인. 미로그인이면 안내 후 중단.
2. **파라미터 파악** — 사용자 요청에서 날짜(기본 오늘), 층, 시간대(start/end), 정원(cap)을 뽑는다.
   층을 특정하지 않았으면 되물어 좁히길 권장(전체는 데이터가 큼). 전체가 필요하면 층별로 나눠 반복.
3. **데이터 가져오기** — `docs/skill-runtime.md`의 "하루 예약 컴팩트" in-page fetch로 해당 층
   예약을 `[[roomCode,'HH:mm','HH:mm'], ...]`로 받는다. `RELOAD`가 반환되면 탭 새로고침 후 1회 재시도.
4. **렌더** — 받은 JSON을 stdin으로 파이프:

   ```bash
   echo '<json>' | node bin/status.mjs <YYYY.MM.DD> --floor <N> [--cap <N>] [--start HH:mm --end HH:mm] --myid <myId>
   ```

   - `--start/--end`를 주면 그 시간대에 비어있는 회의실만 필터한다.
   - 색상 없이 relay하면 인접 예약은 `█/▓` 음영으로 구분된다(자동). 사용자가 터미널에서 직접
     색을 보려면 `--color`.
5. **결과 전달** — 그리드를 그대로 보여주고, 필요하면 "종일가능/가장 빈 방"을 한 줄로 요약한다.
